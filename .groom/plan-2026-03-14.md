## Direction Locked

**Full Elixir/OTP rebuild.** Thinktank v5 replaces 26K LOC of Go plumbing with ~800 LOC of Elixir.
Key decisions:
- Elixir/OTP for subprocess supervision, fault tolerance, graceful shutdown
- LLM-powered router generates perspectives dynamically (PersonaFlow pattern)
- Two modes: quick (parallel API calls) and deep (Pi agent subprocesses)
- MuonTrap for guaranteed subprocess cleanup on parent death
- Kill-safe artifacts with incrementally-updated manifest.json
- Web search (Exa) mandatory for all deep mode agents
- No TUI, no rate limiting, no model registry — pi and OpenRouter handle these

## Research Conducted

### Three Architecture Audits
1. **Harness engineering** — Agent-as-consumer: 5/10. No structured output, no manifest.
2. **Context engineering** — All dimensions poor/none. Dumb pipe: send everything, no filtering.
3. **LLM infrastructure** — Strong plumbing, weak intelligence. No system messages, weak synthesis.

### Pi/Moonbridge Investigation
- Pi handles agent dispatch, model routing, tool execution, session management
- `--no-session --no-skills --tools read,bash,grep,find` for minimal agents
- `PI_CODING_AGENT_DIR` for custom config
- Moonbridge deprecated (not used)

### Multi-Agent Research Patterns
- DMAD (ICLR 2025): Heterogeneous agents outperform identical prompts
- PersonaFlow (ACM DIS 2025): LLM-generated expert personas enhance relevance
- MoA (ICLR 2025): Prompting-based aggregation achieves SOTA
- Debate rounds add latency without reliable quality gains
- Fan-out/fan-in is the optimal pattern for thinktank

### Elixir/OTP Fit
- OpenAI chose Elixir for Symphony (Codex agent orchestration) — same pattern
- MuonTrap solves subprocess cleanup (cgroups on Linux, process groups on macOS)
- Task.async_stream for parallel dispatch
- Supervision trees for fault tolerance
- 3 deps: req, jason, muontrap
